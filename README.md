3. Running Locally
Start the Backend Server:
cd server
npm install
npm start
(Server runs on http://localhost:3000)

Start the React Frontend:
cd client
npm install
npm run dev
(Game runs on http://localhost:5173)

ğŸ“± Building the Native Android App (APK/AAB)
This project uses Capacitor to bridge the web game to a native Android environment.

Build the production web assets:
cd client
npm run build

Sync the built web assets and plugins to the Android project:
npx cap sync android

Open the project in Android Studio to build the signed APK or App Bundle (.aab):
npx cap open android

Note: Ensure your versionCode and versionName are updated in android/app/build.gradle before releasing to the Google Play Console.

ğŸ› Troubleshooting
Mobile Browser Login Fails: Ensure your production domain (e.g., https://www.google.com/search?q=astro-cat.onrender.com) is added to the Authorized domains list in your Firebase Authentication settings.

"Account played on another device" Error: This is the Session Locking system at work. If a session gets stuck during development, manually change the isOnline field to false for that user in your Firestore database.

AdMob Not Showing on Web: The game is designed to run a "Fake 3-second Ad" on web browsers to maintain the user experience. Native AdMob videos will only trigger when running inside the compiled Android App.
